Profile Field Definitions

Field Name
This is the name of the custom field used in the database. It must be unique.
Text Prompt
This is the text that will be displayed next to the fields in forms and display screens. For example, a custom field named "birthdate" might have a prompt of "Birth Date".
Help Text
Enter an optional short help text string to be displayed when the user hovers over the "information" icon in the profile form.
Field Type
Select the type of field to be used. This selection will determin which of the remaining options are displayed.
Field Length
Maximum Length
(Type=text) These values determind the display length and actual maximum length of entries that can be made.
Rows and Columns
(Type=textarea) Enter the number of rows and columns used for the textarea field
Default Value
(Type=text, date, checkbox) Enter the default value for this field. Date = YYYY-MM-DD, Time = HH:MM:SS. Checkbox checked by default.
Values
(Type=select, radio, multi-check) Enter the available values for this field. Radio Buttons, Dropdown: Select the item to be used as the default. Multi-Checkbox: Check each item that should be checked by default.
Time Format
(Type=date) Select the format for time display, either 12- or 24-hour. Select "None" to suppress the time entry and display.
Input Format
(Type=date) Select the format for date inputs, either M/D/Y or D/M/Y. The display of date fields is governed by the system locale configuration.
Format
(Type=date) Enter the format string to be used for the date/time. Refer to the PHP date formats for more information.
Value
(Type=radio, dropdown) This is a string which indicates the possible values for dropdown selections and radio buttons. The format is "value:text;value:text;..." For example, to have two radio buttons labeled "Red" and "Blue", which return values of "R" and "B" respectively, you would enter "R:Red;B:Blue" as the value.

(Type=static) Enter the literal text or HTML that should be displayed for this field.

Enabled
Check this box to enable the field, uncheck to disable. This allows the administrator to prevent some fields from being used without deleting them. (Deleting a custom field also deletes all user data associated with it).
Auto-Generate?
Check this box to have the system automatically generate a value for this field. This may be useful for membership numbers or other serial numbers. The default action is to create a unique glFusion ID number. You can create a function named "CUSTOM_profile_autogen" that takes two parameters: an array of all profile field definitions and values, and the current user ID. Place this function in private/system/lib-custom.php.
Required
Check this box to make the field required. This also implies that the field will be displayed on the user registration page.

Setting a Checkbox-type field as "required" means that it must be checked before the form can be submitted. This may be useful for acknowledging terms and conditions, for example.

Enter at Registration?
Check this box to have the field appear on the user signup page. This is implied if 'required' is selected, but must be explicitly selected for a "Static" field to appear on the signup form.
Show in Profile?
Check this box to have the field appear in the user's public profile.
Span Columns?
Indicates whether the data field should span both columns. Normally, the form fields are displayed as "prompt: input_field". Checking this box supresses the prompt and causes the field to span across the form. (To simply suppress the prompt, leave it blank. The prompt is not required.)
Position After
Select the existing custom field after which this field will appear. New fields appear at the bottom of the list by default. You may change the field order at any time.
Permissions
Permissions have different meanings than normal in the Profile plugin.
  • Owner:
    • Read: The user can view their own profile field. If this is unchecked, the field is hidden from the user.
    • Edit: The user can edit the data in this field by accessing their Account Settings. If unchecked, the field is read-only
  • Group:
    • Read: the specified group can view the field in the user's profile.
    • Edit: If checked, the specified group can edit the profile field in other users' profiles. Use this to allow some users to manage user profile information without becoming full user administrators.
  • Members, Anonymous:
    This indicates whether site members and/or anonymous users can view the profile information in the user's profile.
Important Notes:
  1. These permissions do not affect fields displayed in custom lists. Lists are governed by the permission assigned to the list.
  2. Users with the "profile.admin" privilege can always view and edit all profile fields.
