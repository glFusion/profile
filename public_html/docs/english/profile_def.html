Custom Profile Field Definitions

Field Definitions
Field Name This is the name of the custom field used in the database. It must be unique.
Text Prompt This is the text that will be displayed next to the fields in forms and display screens. For example, a custom field named "birthdate" might have a prompt of "Birth Date".
Help Text Enter an optional short help text string to be displayed when the user hovers over the "information" icon in the profile form.
Field Type Select the type of field to be used. This selection will determin which of the remaining options are displayed.
Field Length
Maximum Length		 (Type=text) These values determind the display length and actual maximum length of entries that can be made.
Default Value (Type=text, date, checkbox) Enter the default value for this field.
  • Text fields: Enter the default text string.
  • Date fields: Enter a SQL-formatted date ("YYYY-MM-DD") and time ("hh:mm:ss") if needed.
  • Checkboxes: Check the box if it is to be checked by default.
  • Radio Buttons, Dropdown: Select the item to be used as the default.
  • Multi-Checkbox: Check each item that should be checked by default.
Values (Type=select, radio, multi-check) Enter the available values for this field.
  • Radio Buttons, Dropdown: Select the item to be used as the default.
  • Multi-Checkbox: Check each item that should be checked by default.
Field Mask (TESTING) (Type=text) Enter the mask string to be used with iMask for this field.
  • "9" represents a digit.
  • "A" represents a letter.
  • "X" represents an alphanumeric character.
Strip Mask Characters? (TESTING) If this box is checked, then mask characters will be stripped from the text. For example, if the mask is "(999) 999-9999" for a telephone number, checking this box causes the data to be stored as "9999999999". If unchecked the data is stored including the mask characters.
Show Time? (Type=date) Check this box if the time selector is to be available on the datepicker.
12 or 24-hour format? (Type=date) Select whether times will be entered in 12-hour or 24-hour format.
Format (Type=date) Enter the format string to be used for the date/time. Refer to the PHP date formats for more information.
Value

(Type=radio, dropdown) This is a string which indicates the possible values for dropdown selections and radio buttons. The format is "value:text;value:text;..." For example, to have two radio buttons labeled "Red" and "Blue", which return values of "R" and "B" respectively, you would enter "R:Red;B:Blue" as the value.

(Type=static) Enter the literal text or HTML that should be displayed for this field.
Enabled Check this box to enable the field, uncheck to disable. This allows the administrator to prevent some fields from being used without deleting them. (Deleting a custom field also deletes all user data associated with it).
Required Check this box to make the field required. This also implies that the field will be displayed on the user registration page.

Setting a Checkbox-type field as "required" means that it must be checked before the form can be submitted. This may be useful for acknowledging terms and conditions, for example.
Enter at Registration? Check this box to have the field appear on the user signup page. This is implied if 'required' is selected, but must be explicitly selected for a "Static" field to appear on the signup form.
Span Columns? Indicates whether the data field should span both columns. Normally, the form fields are displayed as "prompt: input_field". Checking this box supresses the prompt and causes the field to span across the form. (To simply suppress the prompt, leave it blank. The prompt is not required.)
Position After Select the existing custom field after which this field will appear. New fields appear at the bottom of the list by default. You may change the field order at any time.
Permissions Permissions have different meanings than normal in the Profile plugin.
  • Owner:
    • Read: The user can view their own profile field. If this is unchecked, the field is hidden from the user.
    • Edit: The user can edit the data in this field by accessing their Account Settings. If unchecked, the field is read-only
  • Group:
    • Read: the specified group can view the field in the user's profile.
    • Edit: If checked, the specified group can edit the profile field in other users' profiles. Use this to allow some users to manage user profile information without becoming full user administrators.
  • Members, Anonymous:
    This indicates whether site members and/or anonymous users can view the profile information in the user's profile.
Important Notes:
  1. These permissions do not affect fields displayed in custom lists. Lists are governed by the permission assigned to the list.
  2. Users with the "profile.admin" privilege can always view and edit all profile fields.
